The main objective of the software solution is to build a full -fledged mobile application where a user could able to plan their trip with the use of local geo -social knowledge provided by the system.

When it comes to suggesting the best travel companion or a group of companions by identifying the requested user’s preferences, priorities for each factor, and travel destination that is selected at the trip planning stage of the system.

With the help of data collected from the registration form and then the preference profile creation form that is saved in the DB, the system will identify the preferences and relevant deviation values that the user belongs to. According to the given preferences, priority, and deviations for some factors such as age and budget, the system will check the most considerable factor for that age category among the budget, age of the companion, and languages that can speak. By considering the most weighted factor and other factors respectively for that identified age range, the system will suggest the best travel companion or group of companions along with the matching percentage.

When considering the safety assistant component the main objective of the software solution is to suggest the safeness of a place according to user-defined and predefined schemas with the help of data collected from the google API and the user's selection criteria. The system also needs to give us safety mitigation using past resources, which are used by tourist guidance. Along with that when a user inputs a risk they have faced they will be informed of mitigations and also data will be saved in a data store once other users added the destinations they should be informed of the relevant details.

The main objective of the proposed software solution is to suggest the best time to visit tourist attractions places by identifying the crowdedness, weather conditions, open-close hours, and existing user recommendations according to user preference for crowdedness and weather conditions.

The major objective of the user feedback analyzer is to create an automated system to assess visitor input and determine their needs to deliver better service. We assess the traveler feedback from this system and present the most accurate positive feedback in the system feedback part so that future travelers can obtain more accurate information about the online platform as well as the travel experience in Sri Lanka. The sub-objectives can be categorized into four main sections as below according to the system requirements.

Travel buddy suggesting system

To achieve the optimum result of the main objective of suggesting a travel companion for the system, there are several sub-objectives should be acquired.

      • Identifying the buddies who have their destination within the range of 0-30Km s away from the requested user
      • Identifying the most weighted factors from the user data and preferences
      • Identifying the amount of deviation of the age, and the budget preferred by each age category
      • Allocate the most suitable scores based on their identifies preferences and priorities
      • Calculate the total weighted score and percentage for each user
Safety and security assistant system

The sub-objectives of the safety and security assistant system comprise with following main considerations, which are,

      • Identification of safety rating through social organizations near the destination.
      • Derivation of safety matrixes to identify the safety of an area.
      • Identification and prediction of issues, which can be, occur at a selected destination.
      • Selection and recommendation of safety precautions to Inputted issues
      • Selection of destination points to introduce total security for an identified route.

Along with these functional sub-objectives, the system needs to achieve the below architectural and user interface-based objectives as well.

      • Visualization of data to the user in an effective manner
      • Ensure the privacy of the user when storing sensitive data of issues reported.
Best time to visit suggesting system

To achieve the optimum result of the main objective of suggesting the best time to visit the tourist attraction places, several sub-objectives should be acquired.

      • Identifying the nearby location updates according to the user's selected place. (within range of 0-500m)
      • Filter and Categories nearby updates for the last hour and hourly.
      • Identify the probabilities of the crowd, weather, open-close, and user recommendations.
      • Comparing existing user weather updates with weather API.
      • Calculate the total weighted score and percentage.
User feedback and rating analysis system

To reach the main objective of the system, we must ensure that we acquire the following sub-objectives

      • Provide a more clean and accurate feedback section for the consumers to interact.
      • In an online tourism platform, the feedback section of the system is the place where new travelers are looking to gather information regarding the service provided by the system. So it’s a must to maintain a clean, readable, and more accurate feedback section in the system.
      • Provide a dashboard to the admin with the system-analyzed feedback.
      • From this dashboard, the admin can focus on the negative feedback that has been given by the users of the system and report those issues to the relevant sectors.


Main components of the system

      Travel buddy suggestion system
      Predicting the best time to visit tourist attractions
      Identification and providing safety issues and relevant precautions to take.
      Traveler Feedback Analysis

Through the initial planning stage, the user will be suggesting a travel companion based on age and transportation medium. In the best visit time analyzer module, we have identified the best time to visit tourist attractions by analyzing real-time crowd data, weather conditions, and the positive ratings of the attraction. The safety analysis module will do Identification of safety ratings for the selected route through a weightage model and predict threatening areas using historical data analysis. In the user feedback analysis, the system analyzes the reviews, identifies the positivity and negativity of the feedback, and displays them in relevant places. Admin can review all the feedback through the dashboard and interact with the people who gave the feedback to improve the system.
